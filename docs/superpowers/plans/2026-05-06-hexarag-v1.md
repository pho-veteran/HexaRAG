# HexaRAG v1 Implementation Plan Index

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Provide the canonical execution order and plan index for implementing HexaRAG v1 as an AWS-native W4 chat app with a Vite + React frontend, a FastAPI backend, Bedrock + Bedrock AgentCore orchestration, a right-side observability panel, scheduled knowledge base sync, and deployable Terraform infrastructure.

**Architecture:** HexaRAG implementation is split into four focused plans so local workspace setup, runtime behavior, infrastructure, and evaluation can be executed with smaller reviewable checkpoints. Docker Compose remains the default path for local runtime, seeding, tests, and evaluator execution, while Terraform and AWS docs stay isolated in the infrastructure phase.

**Tech Stack:** Docker Compose, Vite, React, TypeScript, Vitest, React Testing Library, FastAPI, Pydantic, Mangum, boto3, PostgreSQL, DynamoDB, Amazon Bedrock, Bedrock AgentCore, Bedrock Knowledge Bases, EventBridge Scheduler, Terraform.
